Welcome to Batch Scripting Tutorial

Batch is a disappointing Microsoft DOS scripting language, common used for simple things. Used only in MS-DOS/Windows operating system. Let's start complaining then.

Introduction to Batch

Batch, first used in MS-DOS, then moved to Windows, is the command-line interpreter in these systems. Created by Microsoft, provides little functionality, and same as Bash/Shell relies on other system programs. While providing so much problems, it's pretty fast & straightforward. The syntax is super weird, but it's like VBS - if you want to use it, you use a cheat scheme all the time.

Writing Your First Batch Script

Let's start with a simple "Hello, World!" script:


@echo off
echo Hello, World!

This script prints "Hello, World!" to the terminal.

Explanation

The @echo off hides all future commands from showing (echo off), and not showing itself (@ char)

Basic Batch Scripting Syntax

Batch scripts are composed of commands, control structures, and variables. They follow a weird syntax and can include comments for documentation.

Variables and Environment

In Batch scripting, you can define and use variables to store data. Environment variables provide information about the environment in which the script is running.


set /p name=Your name: 
echo Hello, %name%!

This script defines a variable "NAME" and prints a greeting using its value.

Control Flow

Bash scripts support various control structures such as if-else, for loops, and while loops for decision making and looping.

Functions

Batch... doesn't have functions. You must use goto.


:start
set /p whether=Want to be greeted (yes/no)? 

if %whether%==yes goto greet
if %whether%==no goto nogreet
echo This is not a correct answer bro.
goto start

:greet
echo Hello!
goto end

:nogreet
echo Uhh, ok.
goto end

:end
pause
exit

File Operations

Bash scripts can perform various file operations such as creating, reading, writing, and deleting files using built-in commands and utilities.


#!/bin/bash
if [[ $1 == "-h" ]]; then
    echo "Available commands:"
    echo " $0 del - deletes the existing note"
    echo " $0 edi - edits the existing note using Vim"
    exit 0
fi

if [[ $1 == "del" ]]; then
    rm .note
    echo "Your note has been deleted."
    exit 0
fi

if [[ $1 == "edi" ]]; then
    vim .note
    exit 0
fi

if [[ -f ".note" ]]; then
    echo "Your note:"
    cat .note
else
    echo ================ > .note
    echo Name: New Note >> .note
    echo Tags: >> .note
    echo ================ >> .note
    vim .note
    echo "Your note has been saved."
fi

This is a notes script written in Bash, I've created right now.

Variables used

$0 is the script path, $1 is the first argument, and it's going like that.

Commands used

echo prints text, vim is a file editor, rm is used for deleting files, cat is used for viewing files, and exit 0 is used for exiting the program, with no error (0 is succesful).

Command-Line Arguments

Bash scripts can accept command-line arguments passed to them when they are executed. These arguments can be accessed within the script using special variables.

Error Handling

Bash scripts can handle errors using conditional statements and error-handling mechanisms to ensure robustness and reliability.
